Help

Login page ^top

Welcome to the B2NOTE annotation service.

B2NOTE is currently designed as a crowdsourcing service which implies that all the annotations are accessible publicly. To guarantee on one side your privacy and on the other side to keep track of the origin of annotation records, you must first register to create an annotator profile with a pseudonym which will be used to publish your annotation.

This page is the entry point to access the service and allows you either register as first time user (2) or log in directly if you already created an account (1). In case you forgot your password, you can re-initialize it by clicking on the link “Forgot password” (3)

Registration^top

Fill in the required information to create an annotator profile and do not forget to enter the captcha before submitting.


If you forgot to fill-in some information or provided an invalid email address, a red error message will appear above the field to correct.

Upon successful registration you should be redirected to the Login page.

Main page ^top

Create and sum-up annotations.

Why annotating? Using B2NOTE you can add and manage extra information (“annotations”) about data files hosted by a EUDAT service (a B2SHARE training instance during the pilot phase). Then use the annotations for finding files of interest and identifying new user-specific datasets.

This is the B2NOTE service main page. It contains the following elements:

  1. Navigation bar: access different functionalities of the B2NOTE service.
  2. Main panel: create an annotation.
  3. “All my annotations”: visualise a summary of your annotations and access more detailed information.
  4. “All annotations about this file”: visualise a summary of the annotations about the current file and access more detailed information.
  5. Feedbacks button: assess the service, request a new feature or report a bug.
Navigation bar

  • User account: access your annotator profile information.
  • Search: find files using annotations and export the resulting datasets.
  • Export: export annotations in JSON-LD (linked-data) format.
  • Logout: log out of the B2NOTE service, terminates the current user session.
  • Help: access the help section for page you are visiting.
Navigation bar

At the top of other pages additional shortcut links allow quick navigation to previous pages or straight back to the B2NOTE console main page.

  • Previous: previously accessed page.
  • Home: the B2NOTE console main page.
Main panel

Create annotations.

The main panel is divided-up into 3 tabs. Each tab contains a simple form for creating an annotation of one of the 3 types defined in B2NOTE.

  1. Semantic tag: for creating an annotation that uses a term defined in a controlled vocabulary as a tag.
  2. Free-text keyword: for creating an annotation that uses your own keyword as a plain-text tag.
  3. Comment: for creating an annotation with longer pieces of plain-text.
Creating a semantic tag annotation

The semantic tag annotations use a term defined in a controlled vocabulary as a tag. This type of annotations thus benefits from curated and uniquely identified linked-data material.

Clicking the text-area then typing-in the first letters of the term you want triggers the autocomplete functionality, which provides a list of suggestions.

From the list of suggestions, you can then select your term of interest amongst the extensive B2NOTE index.

After selecting one of the suggestions, click the “Create” button to create the annotation.

Creating a free-text keyword annotation

Free-text keyword annotations are designed for cases when the exact term you want is missing from the “Semantic tag” autocomplete list of suggestions and you want a plain-text tag to be used in the annotation. Annotations of this type contain plain text information as opposed to linked-data, thus not allowing the full range of benefits provided by the “Semantic tag” type.

After typing the text for your custom keyword in the text-area, click the “Create” button to create the annotation.

Creating a comment annotation

Comment annotations are designed for annotating using longer pieces of text. That is, possibly more verbose text not intended for the purpose of tagging a document, such as commenting or leaving some evaluating or general quality feedback. Annotations of this type also, like the “free-text keyword” type, contain plain text information as opposed to linked-data, thus not allowing the full range of benefits provided by the Semantic tag” type.

After typing-in the text of the comment, click the "Create" button to create the annotation.

After selecting a semantic tag or typing-in a free-text keyword or comment text, click the “Create” button to create the annotation.


Some conditions prevent annotations from being created.

  1. Duplicate: it is not possible to create an annotation that would be a duplicate of an existing annotation. So a semantic tag or free-text tag annotation can not use the same tag about the same file as another annotation. When attempting to do so, an alert will be displayed informing you that this is the case.

  2. Semantic equivalent: attempting to create a free-text keyword annotation using a label identical to an ontology term referenced by B2NOTE, triggers an alert asking you to confirm the type of annotation you want to create:

    • Click the “Semantic” button to select a semantic tag instead of your free-text keyword.
    • Click the “Keyword” button to confirm creating an annotation using your free-text keyword.
    • Click the “Cancel” button to cancel the annotation creation.

  3. Long keyword: attempting to create a free-text keyword annotation with a tag label longer than 60 characters will trigger an alert asking to confirm the type of annotation you want to create:

    • Click the “Keyword” button for an annotation of the Free-text Keyword type.
    • Click the “Comment” button for an annotation of the Comment type.
    • Click the “Cancel” button to cancel the annotation creation.
“All my annotations” section

Click the leftmost chevron icon (circled in red above) to unfold a summary table with the number of annotations of each type that were created by you.

  • Click the annotation type links or numbered badge of each row to a summary table with more detailed information about corresponding annotations.
  • Click the All my annotations” section title to the page containing all the summary tables about annotations created by you.
“All annotations about this file” section

Click the leftmost chevron icon (circled in red above) to unfold a summary table with the number of annotations about the current file of each type that were created by other users (blue badges) and those that were created by you (orange badges).

  • Click each annotation type link or numbered badge to a summary table with more detailed information about corresponding annotations.
  • Click the “All annotations about this file” section title to the page containing all summary tables for annotations created about the file.
Feedbacks button

“Let us know what you think” button: access forms designed for providing us an assessment about B2NOTE, requesting a new feature or reporting a bug.

My annotations page ^top

Review and edit your existing annotations.

Access this page from the B2NOTE console main page by clicking the “All my annotations” link.

  1. Summary table: showing how many annotations of each type were created by you and linking to the corresponding detailed information tables.

  2. Detailed information table: on this page there are 3 such tables, one for each type of annotation. They show the labels of corresponding annotations that were created by you, link to individual annotation data and links for editing or deleting each annotation.

Summary table

Referencing the number of annotations of each type that were created by you.

Click each link or numbered badge to a table containing more detailed information about annotations of the corresponding type (see below).

Detailed information table

There is one such table per annotation type (Semantic tag, Free-text keyword and Comment). Each row of the table features:

  1. Label of the annotation: click to access detailed information about the tag.

  2. Numbered badge: reporting how many of your annotations use the same tag. Click to access the corresponding list of files and detailed information about the tag. There are no numbered badges in the Comment annotation table since these annotations are by definition unique.

  3. Management options:

    • Click a pen icon to edit the corresponding annotation.
    • Click a bin icon to delete the corresponding annotation.

All annotations page ^top

Review existing annotations about a file.

Access this page from the B2NOTE console main page by clicking the “All my annotations” section title.

The functionalities available from this page of the B2NOTE console are detailed below.

Summary table: referencing the number of annotations of each type that were created about a file, either by other users (blue badges) or yourself (orange badges). Clicking each link or numbered badge redirects to a table containing more detailed information about annotations of the corresponding type (see below).

Detailed information tables: reporting for each annotation type (Semantic tag, Free-text keyword or Comment), the labels of the annotations about a file, and - except for comment annotations - the number of times an identical tag was used in other annotations in successive rows. Tag reuse by other users is reported in blue badges, while tag reuse in your annotations is reported in orange badges. Each row also features annotation management options for editing (pen icon) or deleting (bin icon) the corresponding annotation. These options are only available (clickable blue icons) when the annotation was created by you otherwise they are inactive (pale grey icons). The tag label link can be clicked to access more detailed information about a particular tag, such as the full text of the tag label, and information about the original ontology term in the case of semantic tag annotations. Clicking one of the numbered badges directs to a page listing the different files annotated using a particular tag.

Edit annotation page ^top

Change an annotation.

Access to this page by clicking one of the pen icons (circled in red above) associated to annotations created by you in tables of the “My annotations” or “All annotations” summary pages.

The functionalities available from this page of the B2NOTE console are detailed below.

Edit menu: allows the tag of an individual annotation to be modified.

Typing inside the text-area allows a different semantic tag to be selected amongst the suggestions for replacing the existing tag (in case of editing a semantic tag annotation) or modifying the text of free-text annotations (i.e. keyword or comment types).

  • Click the “Save” button for enforcing the modified tag as the new tag.
  • Click the “Cancel” button to avoid changing the original tag.

The process of modifying the annotation tag is very similar to that of creating the tag for a new annotation as performed from the main page interface. The same restrictions and alerts apply. In particular it is not allowed to generate an annotation that would be a duplicate (same tag about the same file) of another existing annotation by editing it.
