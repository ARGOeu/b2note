Help

Login page ^top

Welcome to the B2NOTE annotation service.

B2NOTE is currently designed as a crowdsourcing service which implies that all the annotations are accessible publicly. To guarantee on one side your privacy and on the other side to keep track of the origin of annotation records, you must first register to create an annotator profile with a pseudonym which will be used to publish your annotation.

This page is the entry point to access the service and allows you either register as first time user (2) or log in directly if you already created an account (1). In case you forgot your password, you can re-initialize it by clicking on the link “Forgot password” (3)

Registration page ^top

Create a B2NOTE user account.

This page displays a form to fill-in with the personal information required to create a B2NOTE user account allowing subsequent logging-in to the service.

All the fields need to be filled-in and the captcha correctly identified for the registration process to complete upon clicking the “Submit” button.

Main page ^top

Create and sum-up annotations.

Why annotating? B2NOTE allows defining user-specific information about online resources such as data files hosted on a EUDAT partner service. Custom new user-specific datasets can thus be constituted through the process of adding some particular information about multiple files.

The functionalities available from the main page of B2NOTE console are detailed below.

Navigation bar

  • User account (head outline icon with pseudonym link): page displaying the data filled-in during registration (user-specific information).
  • Search (magnifying glass icon): find files based the content of annotations and export the resulting information dataset.
  • Export (arrow down to inbox icon): export annotations in JSON-LD (linked-data) format.
  • Logout (arrow pointing right out of box icon): logging-out from the B2NOTE service, terminates the current user session.
  • Help (interrogation mark icon): directs to the help section specific for the calling page.

At the top of other pages additional shortcut links allow quick navigation to previous pages or straight back to the B2NOTE console main page.

  • Arrow pointing left: clicking redirects to the previously accessed page.
  • House outline: clicking always redirects directly to the B2NOTE console main page.
Main panel

Create annotations.

The main panel is divided-up into 3 tabs. Each tab contains a simple form for creating an annotation of one of the 3 types defined in B2NOTE.

“Semantic tag” tab (selected by default): for creating an annotation that uses a term defined in a controlled vocabulary as a tag. This type of annotation thus benefits from curated and uniquely identified linked-data material. Clicking the text-area then typing-in the first letters of the tag you want triggers the autocomplete functionality, which provides a list of matching suggestions.

From this list of suggestions, you can select your term of interest amongst the B2NOTE extensive index. Currently the B2NOTE index is built from over 400 Bioportal ontologies covering a range of knowledge domains.

After selecting one of the suggestions, clicking the create button will trigger an annotation record to be created.

“Free-text keyword” tab: for using your own free-text keyword as a tag. This option is designed for cases when the exact term you want is missing from the “Semantic tag” autocomplete list of suggestions, or plain-text is purposefully intended to be used in the annotation. However, annotations that are not created through the “Semantic tag” option will contain plain text information and will not provide the full range of benefits allowed by curated and uniquely identified linked-data material.

After typing-in a free-text keyword, clicking the create button will trigger an annotation record to be created.

“Comment” tab: for using longer pieces of text as annotations. That is, possibly more verbose text not intended for the purpose of tagging a document, such as commenting or leaving some evaluating or general quality feedback. Annotations of this type also, like the “free-text keyword” type, contain plain text information as opposed to linked-data. Thus not providing the full range of benefits allowed by curated and uniquely identified linked-data material.

After selecting a semantic tag or typing-in a free-text keyword or comment text, clicking the create button will trigger an annotation record to be created.

Some conditions preclude annotations from being created.

It is not possible to create an annotation that would be a duplicate of an existing annotation. So a semantic tag or free-text keyword annotation cannot use the same tag about the same file as another annotation. When attempting to do so, an alert will be displayed providing information as to the reason why the annotation could not be created.

Attempting to create a free-text keyword annotation using a label identical to an ontology term referenced by B2NOTE, triggers an alert. This alert allows you to decide to use the semantic tag instead (click the “Semantic” button) or confirm your intent of creating a free-text keyword annotation (click the “Keyword” button). The alert menu also offers a button for cancelling the process of creating this particular annotation (click the “Cancel” button).

Attempting to create a free-text keyword annotation with a tag label longer than 60 characters will trigger an alert asking to confirm your intent of creating an annotation of the free-text keyword type (click the “Keyword” button) and offering to create an annotation of the type “comment” instead, while keeping your text identical (click the “Comment” button). The alert menu also offers a button for cancelling the process of creating this particular annotation (click the “Cancel” button).

“All my annotations” section: clicking the leftmost chevron icon (circled in red above) unfolds a summary table referencing the number of annotations of each type that were created by you.

Clicking a numbered badge or annotation type link redirects to a summary table containing more detailed information about the corresponding annotations and annotation management options. Clicking the section title link (“All my annotations”) loads the page containing all the summary tables about annotations created by you.

“All annotations about this file” section: clicking the leftmost chevron icon (circled in red above) unfolds a summary table referencing the number of annotations about a file of each type that were created by other users (blue badges) and those that were created by you (orange badges).

Clicking each numbered badge or annotation type link redirects to a summary table containing more detailed information on corresponding annotations and annotation management options. Clicking the section title link (“All annotations about this file”) loads the page containing all summary tables for annotations created about a file.

Feedbacks button

“Let us know what you think” button: click to access forms designed for providing us your assessment about B2NOTE, requesting a new feature or reporting a bug.

My annotations page ^top

Review and edit your existing annotations.

Access to this page from the B2NOTE console main page by clicking the “All my annotations” link.

The functionalities available from this page of the B2NOTE console are detailed below.

Summary table: referencing the number of annotations of each type that were created by you. Clicking each link or numbered badge redirects to a table containing more detailed information about annotations of the corresponding type (see below).

Detailed information tables: reporting for each annotation type (Semantic tag, Free-text keyword or Comment), the labels of the annotations, and the number of times an identical tag was used in any of your annotations, in successive rows. Each row also features annotation management options for editing (pen icon) or deleting (bin icon) the corresponding annotation. The tag label link can be clicked to access more detailed information about a particular tag, such as the full text of the tag label, and information about the original ontology term in the case of semantic tag annotations. Clicking one of the numbered badges directs to a page listing the files annotated using a particular tag.

All annotations page ^top

Review existing annotations about a file.

Access to this page from the B2NOTE console main page by clicking the “All annotations about this file” link.

The functionalities available from this page of the B2NOTE console are detailed below.

Summary table: referencing the number of annotations of each type that were created about a file, either by other users (blue badges) or yourself (orange badges). Clicking each link or numbered badge redirects to a table containing more detailed information about annotations of the corresponding type (see below).

Detailed information tables: reporting for each annotation type (Semantic tag, Free-text keyword or Comment), the labels of the annotations about a file, and - except for comment annotations - the number of times an identical tag was used in other annotations in successive rows. Tag reuse by other users is reported in blue badges, while tag reuse in your annotations is reported in orange badges. Each row also features annotation management options for editing (pen icon) or deleting (bin icon) the corresponding annotation. These options are only available (clickable blue icons) when the annotation was created by you otherwise they are inactive (pale grey icons). The tag label link can be clicked to access more detailed information about a particular tag, such as the full text of the tag label, and information about the original ontology term in the case of semantic tag annotations. Clicking one of the numbered badges directs to a page listing the different files annotated using a particular tag.

Edit annotation page ^top

Change an annotation.

Access to this page by clicking one of the pen icons (circled in red above) associated to annotations created by you in tables of the “My annotations” or “All annotations” summary pages.

The functionalities available from this page of the B2NOTE console are detailed below.

Edit menu: allows the tag of an individual annotation to be modified.

Typing inside the text-area allows a different semantic tag to be selected amongst the suggestions for replacing the existing tag (in case of editing a semantic tag annotation) or modifying the text of free-text annotations (i.e. keyword or comment types).

  • Click the “Save” button for enforcing the modified tag as the new tag.
  • Click the “Cancel” button to avoid changing the original tag.

The process of modifying the annotation tag is very similar to that of creating the tag for a new annotation as performed from the main page interface. The same restrictions and alerts apply. In particular it is not allowed to generate an annotation that would be a duplicate (same tag about the same file) of another existing annotation by editing it.
